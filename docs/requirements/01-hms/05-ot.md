# Operation Theatre Scheduling & Management

| Field | Value |
|---|---|
| Module | HMS |
| Sub-module | OT |
| Status | Draft — pending verification |
| Source | §3.1 OT; §3.2 step 6 |

## 1. Scope
Theatre scheduling, pre-op checklist, intra-op documentation hooks, and implant/consumable usage logging against patient.

## 2. Exclusions
Full anesthesia information system depth in later phases; robotic device telemetry.

## 3. Actors and Permissions
- OT Scheduler
- Surgeon
- Anesthetist
- OT Nurse

## 4. Functional Requirements
| ID | Requirement |
|---|---|
| HMS-OT-FR-001 | Schedule OT cases with theatre, team, and equipment requirements. |
| HMS-OT-FR-002 | Enforce pre-op checklist completion before start. |
| HMS-OT-FR-003 | Record procedure start/end and team timeouts. |
| HMS-OT-FR-004 | Log implants/consumables used and send to Inventory + Billing. |
| HMS-OT-FR-005 | Capture post-op destination (recovery/ICU/ward). |

## 5. Workflow and State Transitions
OT Case: `Requested -> Scheduled -> PreOp -> InProgress -> Completed | Cancelled`.
Checklist: `Pending -> Completed`.

## 6. Data / Entities and Validation
- `OtCase`
- `OtChecklist`
- `OtImplantUsage`
- `OtTeamAssignment`

## 7. Business Rules
| ID | Rule |
|---|---|
| HMS-OT-BR-001 | Surgeon/anesthetist must have valid specialty privileges. |
| HMS-OT-BR-002 | Implant usage requires batch/serial capture for traceable items. |
| HMS-OT-BR-003 | Overlapping exclusive theatre bookings forbidden. |

## 8. Approvals
Case cancellation after consumable reservation may require supervisor approval.

## 9. APIs and Module Ownership
**Owner:** HMS

### APIs
- `POST /api/hms/ot/cases`
- `POST /api/hms/ot/cases/{id}/checklist`
- `POST /api/hms/ot/cases/{id}/usage`
- `POST /api/hms/ot/cases/{id}/complete`

### Events Published
- `OtCaseScheduled`
- `OtCaseCompleted`
- `OtImplantUsed`

### Events Consumed
- `StockAvailabilityChanged`
- `EmployeeCredentialStatusChanged`

## 10. Notifications
- Schedule changes to OT team
- Missing checklist blockers

## 11. Reports
- OT utilization
- Case cancellation reasons
- Implant usage register

## 12. Audit, Retention, and Privacy
Surgical records and implant registers retained per clinical device regulations; immutable usage log.

## 13. Failure, Idempotency, and Concurrency
- Theatre schedule conflict detection transactional
- Usage posting idempotent

## 14. Non-Functional Requirements
- Schedule board consistent across concurrent editors

## 15. Dependencies
- HMS IPD/EMR/Billing
- INV stock
- HRIS privileges
- FIN charges/assets optional

## 16. Acceptance Criteria
| ID | Criterion |
|---|---|
| HMS-OT-AC-001 | Completing OT case with implants decrements stock and adds bill charges. |
| HMS-OT-AC-002 | Case cannot start with incomplete mandatory checklist. |

## 17. Open Assumptions
- Checklist templates configurable per procedure type.
