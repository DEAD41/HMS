# Accounts Payable

| Field | Value |
|---|---|
| Module | FIN |
| Sub-module | AP |
| Status | Draft — pending verification |
| Source | §5.1 AP; §5.2 B procure-to-pay; three-way match |

## 1. Scope
Vendor invoice capture, three-way match against PO/GRN, payment scheduling, and GL posting.

## 2. Exclusions
Dynamic discounting marketplace.

## 3. Actors and Permissions
- AP Clerk
- AP Supervisor
- Finance Controller

## 4. Functional Requirements
| ID | Requirement |
|---|---|
| FIN-AP-FR-001 | Capture vendor invoices against vendors/POs. |
| FIN-AP-FR-002 | Perform three-way match (PO, GRN, invoice) with tolerance rules. |
| FIN-AP-FR-003 | Schedule and execute payments per terms. |
| FIN-AP-FR-004 | Post AP and payment entries to GL/vendor ledger. |
| FIN-AP-FR-005 | Manage debit notes and adjustments. |

## 5. Workflow and State Transitions
VendorInvoice: `Received -> Matching -> Matched | Variance -> Approved -> Scheduled -> Paid | Void`.
Payment: `Proposed -> Approved -> Disbursed | Failed`.

## 6. Data / Entities and Validation
- `VendorInvoice`
- `MatchResult`
- `PaymentProposal`
- `ApPayment`

## 7. Business Rules
| ID | Rule |
|---|---|
| FIN-AP-BR-001 | Payment blocked unless matched/approved or explicit exception. |
| FIN-AP-BR-002 | Tolerances configurable by item/vendor/category. |

## 8. Approvals
Payment release uses FND approvals by amount threshold.

## 9. APIs and Module Ownership
**Owner:** FIN

### APIs
- `POST /api/fin/ap/invoices`
- `POST /api/fin/ap/invoices/{id}/match`
- `POST /api/fin/ap/payments`

### Events Published
- `VendorInvoiceMatched`
- `ApPaymentDisbursed`

### Events Consumed
- `PurchaseOrderIssued`
- `GoodsReceived`
- `VendorChanged`

## 10. Notifications
- Match variance alerts
- Payment due reminders

## 11. Reports
- AP aging
- Three-way match exceptions

## 12. Audit, Retention, and Privacy
AP documents retained; vendor banking data confidential.

## 13. Failure, Idempotency, and Concurrency
- Match/payment idempotent
- Payment disbursement exactly-once effective

## 14. Non-Functional Requirements
- Match engine for typical invoice < 1s

## 15. Dependencies
- PRC PO
- INV GRN
- FIN GL
- FND Approvals

## 16. Acceptance Criteria
| ID | Criterion |
|---|---|
| FIN-AP-AC-001 | Exact PO+GRN+invoice match allows payment scheduling. |
| FIN-AP-AC-002 | Duplicate invoice number per vendor rejected. |

## 17. Open Assumptions
- OCR invoice capture optional later.
